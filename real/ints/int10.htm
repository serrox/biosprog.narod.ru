INT 10h: Видео сервис

Здесь детализированы функции INT 10h стандартного видеосервиса ROM-BIOS.

AH сервис
00h Устанавливает видео режим. Очистить экран, установить поля BIOS, установить режим. вход: AL=режим
AL Тип формат цвета адаптер адрес примечания
0* текст 40x25 2 CGA,EGA,VGA,SVGA+ b8000
1 текст 40x25 16 CGA,EGA,VGA,SVGA+ b8000
2* текст 80x25 2 CGA,EGA,VGA,SVGA+ b8000
3 текст 80x25 16 CGA,EGA,VGA,SVGA+ b8000
4 графика 320x200 4 CGA,EGA,VGA,SVGA+ b8000
5* графика 320x200 4 полутона CGA,EGA,VGA,SVGA+ b8000
6 графика 640x200 2 CGA,EGA,VGA,SVGA+ b8000
7 текст 80x25 3 (b/w/bold) MA,EGA,VGA,SVGA+ b0000
0Dh графика 320x200 16 EGA,VGA,SVGA+ A0000
0Eh графика 640x200 16 EGA,VGA,SVGA+ A0000
0Fh графика 640x350 3 (b/w/bold) EGA,VGA,SVGA+ A0000
10h графика 640x350 16 EGA,VGA,SVGA+ A0000
11h графика 640x480 2 MCGA,EGA,VGA,SVGA+ A0000
12h графика 640x480 16 EGA,VGA,SVGA+ A0000
13h графика 320x200 256 MCGA,VGA,SVGA+ A0000
6Ah графика 800x600 16 SVGA+ (VESA VBE совместимая) A0000

поддерживается не всеми BIOS!

* при установке этих режимов отключён механиз преобразования палитры.

На самом деле известных режимов великое множества, но большинство из них поддерживается только на некоторой конкретной аппаратной платформе или даже при наличии какой-то конкретной программы в памяти, поэтому я решил не описывать их здесь все.

Замечание: SVGA+ обозначает модели как с поддержкой SVGA, так и более новых технологий
Замечание:если бит 7 регистра AL = 0, то экран очищается, в обратном случае (AL. 7=1) содержимое экрана не изменяется.
Для установки режимов VESA BIOS Extension испульзуйте функцию 4Fh подфункцию 2 прерывания 10h
01h Установить размер/форму курсора в текстовом режиме. Курсор, если он видим, всегда мерцает.

вход: CH = начальная строка (0-1Fh; 20h=подавить курсор)
CL = конечная строка (0-1Fh)

Стандартно экран содержит 25 строк, однако это значение меняется в зависимости от режима.
По умолчанию ch=13 cl=14

В графических режимах курсор нужно моделировать. Для этого можно использовать ASCII символ с кодом 0DFh (сплошной прямоугольник)
02h Устанавливает позицию курсора.

вход: BH = видео страница
DH = номер строки (считая от 0)
DL = номер столбца (считая от 0)

Установка номера страницы производится при помощи функции 5h прерывания 10h
03h Узнать позицию и размер курсора

вход: BH = видео страница
выход: DH = текущая строка курсора (см. функцию 02H)
DL = текущая колонка курсора (см. функцию 02H)
CH = текущая начальная строка курсора (см. функцию 01H)
CL = текущая конечная строка курсора (см. функцию 01H)
04h Читать световое перо

вход: нет
выход: AH = триггер (0=нет значений; 1=возвращены значения светового пера)
DH,DL = строка,колонка символа (текст)
BX = колонка точки (графика)
CH = строка точки (для графики EGA возвращается в CX)
05h Выбрать активную страницу дисплея

вход: AL = номер страницы (большинство программ использует страницу 0)
06h Листать окно вверх (или очистить). Листать на 1 или более строк вверх.

вход: CH,CL = строка,колонка верхнего левого угла окна (считая от 0)
DH,DL = строка,колонка нижнего правого угла окна (считая от 0)
AL = число пустых строк, вдвигаемых снизу (0=очистить все окно)
BH = видео атрибут, используемый для пустых строк
07h Листать окно вниз (вдвинуть пустые строки в верхнюю часть окна)

вход: (аналогично функции 06h)
08h Читать символ и атрибут в текущей позиции курсора

вход: BH = номер видео страницы
выход: AL = прочитанный символ (ASCII код)
AH = прочитанный видео атрибут (только для текстовых режимов)
09h Писать символ/атрибут в текущей позиции курсора

вход: BH = номер видео страницы
AL = записываемый символ (ASCII код)
CX = счетчик (сколько экземпляров символа записать)
BL = видео атрибут (текстовый режим) или цвет (графический режим)

При записи с помощью этой функции курсор не сдвигается!
0Ah писать символ в текущей позиции курсора
вход: BH = номер видео страницы, для видео режима 13h - цвет фона
AL = записываемый символ(ASCII код)
CX = счетчик (сколько экземпляров символа записать)
0Bh выбрать цвет палитры/бордюра (CGA-совместимые режимы)
вход:
BH = 0 (текст) выбрать цвет бордюра
BL = цвет бордюра (0-1Fh; от 10H до 1Fh - интенсивные)
BH = 1 (графика) выбрать палитру
BL = 0: палитра green/red/brown
BL = 1: палитра cyan/magenta/white
BH=2 Только для Tandy2000
установить палитру
не думаю, что имеет смысл здесь описывать эту функцию
0Ch писать графическую точку (слишком медленно для большинства приложений!)

вход: BH = номер видео страницы
DX,CX = строка,колонка или координаты x и y
AL = значение цвета

При установленом бите 7 регистра AL точка будет иметь цвет, полученый в результате поразрядной операции XOR со стрым чветом точки с координатами CX,DX
0Dh читать графическую точку (слишком медленно для большинства приложений!)
Только графический режим

вход: BH = номер видео страницы
DX,CX = координаты x,y
выход: AL = прочитанное значение цвета
0Eh писать символ на активную видео страницу (эмуляция телетайпа)

вход: AL = записываемый символ (использует существующий атрибут)
BL = цвет переднего плана (для графических режимов)

Символы CR(0Dh) LF(0Ah) BELL(7) интерпритируются как управляющии. Атрубут устанавливается равным атрибуту текущего символа.
0Fh читать текущий видео режим

выход: AL = текущий режим (см. функцию 00h)
AH = число текстовых колонок на экране
BH = текущий номер активной страницы дисплея
10h Установить регистры палитры для EGA. Работает при наличии EGA-совместимой карты.(включая VGA-совместимые)

Выбирает цвета, используемые данным атрибутом экрана. умалчиваемые цвета те же, что используются с CGA (см. атрибуты экрана ).

вход:
AL = 0 уст. один регистр палитры
BL = регистр (идентифицирует 4-битовый цвет; 0-15)
BH = значение регистра (6-битовый rgbRGB выдаваемый цвет)
AL = 1 уст. регистр рамки.
BH = значение регистра (6-битовый rgbRGB выдаваемый цвет)
AL = 2 уст. все регистры палитры и регистр бордюра
ES:BX => 17 байт (регистры палитры (биты 0-15) и регистр бордюра (бит 16))
AL = 3 выбрать яркость или мерцание
BL,бит7 = 0:яркость фона (16 возможных цветов фона)
BL,бит7 = 1: мерцание (8 цветов фона + мерцание переднего плана)
AL = 7Чтение регистра палитры
необходима VGA-совместимая видеокарточка
На входе: BL = номер читаемого регистра палитры (0-15).
На выходе: BH = содержимое регистра палитры.
AL = 8Чтение регистра цвета рамки
необходима VGA-совместимая видеокарточка
На выходе: BH = содержимое регистра цвета рамки

На выходе: По адресу ES:DX содержится таблица, заполненная значениями из регистров палитры

необходима VGA-совместимая видеокарточка
BX = номер регистра таблицы цветов (0-255);

На входе: BL = номер первого регистра таблицы цветов (0-255);
